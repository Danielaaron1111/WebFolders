Demo 4: position: fixed

Element is fixed relative to the viewport. Stays in place when scrolling.

Example 4.1: Fixed Element Demo

.fixed-box { position: fixed; bottom: 20px; right: 20px; }
/* Stays in viewport even when scrolling */

üìå Look at the bottom-right corner of your screen! There's a fixed "Back to Top" button. It stays there even when you scroll. That's position: fixed in action!

Common Fixed Positions:

  • top: 0; left: 0; - Top-left corner
  • top: 0; right: 0; - Top-right corner
  • bottom: 0; left: 0; - Bottom-left corner
  • bottom: 0; right: 0; - Bottom-right corner
  • top: 0; left: 0; right: 0; - Full-width header
  • bottom: 0; left: 0; right: 0; - Full-width footer

Example 4.2: Fixed vs Absolute

position: absolute

  • Positioned relative to nearest positioned ancestor
  • Scrolls with the page
  • Removed from document flow

position: fixed

  • Positioned relative to viewport
  • Stays in place when scrolling
  • Removed from document flow

üîÑ Both absolute and fixed are removed from the flow, but fixed always positions relative to the viewport, while absolute positions relative to the nearest positioned ancestor.

Demo 5: position: sticky

Hybrid of relative and fixed. Element is relative until scroll threshold, then becomes fixed.

Example 5.1: How Sticky Works

.sticky-element {
position: sticky;
top: 0; /* Threshold: sticks when it reaches top */
}

üé™ The navigation bar at the top of this page is sticky! It scrolls normally until it reaches the top, then "sticks" there. Scroll up and down to see it in action.

Sticky Behavior Stages:

  1. Stage 1: Element scrolls normally (behaves like position: relative)
  2. Stage 2: When threshold is reached (e.g., top: 0), element "sticks"
  3. Stage 3: Element stays fixed until parent container scrolls past
  4. Stage 4: Element scrolls away with its parent

Position Properties Cheat Sheet

static

  • ‚úÖ Default value
  • ‚úÖ Follows normal flow

  • ‚úÖ Offset from normal position
  • ‚úÖ Original space preserved
  • ‚úÖ Can use top/right/bottom/left
  • ‚úÖ Creates positioning context

  • ‚úÖ Removed from flow
  • ‚úÖ Positioned to nearest ancestor
  • ‚úÖ Can use top/right/bottom/left

  • ‚úÖ Removed from flow
  • ‚úÖ Positioned to viewport
  • ‚úÖ Stays during scroll
  • ‚ùå No space reserved
